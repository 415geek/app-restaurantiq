## 信息架构（Pass 1）— 可交付结构

### 漏斗阶段（与 PRD 三步骤对齐并补强）

PRD 草案：`type → address → tier`。上线建议固定为 **4 个心理步骤**（可在 UI 上合并为 3 屏）：

1. **承诺与对标**（首屏 5 秒内）  
   - 主标题：**业态中立**（非仅餐厅），例如「这个位置适合开这家店吗？」+ 副标保留 PRD 价值句之一（二选一 A/B 测）。  
   - **信任条**（非长表格）：一行字 + 3 个 logo/名称：`Placer.ai · SiteZeus · Buxton` + 「同类洞察，单次 $29 起」——避免 PRD 中大表格占首屏。

2. **业态选择**  
   - **大类**（7）→ **子类型**（每类 3–5）：二级面板或抽屉，避免首屏 30+ 选项轰炸。  
   - **默认选中**：`restaurant / chinese_restaurant` 或与 UTM `?vertical=retail` 联动，减少点击。

3. **地址**  
   - 主输入 +（可选）「与地图确认」：减少错误地址导致的退款与差评。  
   - **占位与错误**：空提交、地址过短、地理编码失败 — 需在 PRD 中写清**用户看到的句子**（非仅 500）。

4. **档位与支付前确认**  
   - **默认档**：PRD 写默认 `pro` — 与「最受欢迎」锚定一致，**可交付**：中间列 `Recommended` 徽章 + 略大阴影（具体 token 见下）。  
   - **支付前一行摘要**：`{业态} · {地址截断} · {档位}` +「分析约需 N 秒」降低弃单。

### 首屏视线顺序（必须写进研发说明）

| 功能 | LOADING | EMPTY | ERROR | SUCCESS | PARTIAL |

| 地址 | Google/Maps 联想 loading | 占位提示 | 无效地址 / 超出服务区域 | 有效 → 可下一步 | 模糊匹配时列出 3 条备选 |

## 用户旅程与情绪（Pass 3）

| 步骤 | 用户动作 | 期望情绪 | 设计支撑 |
|------|-----------|----------|----------|
| 落地 | 扫码/广告进入 | 「这像正经产品，不是玩具」 | 去掉 PRD 示例里 **标题 emoji（🎯）**；避免 `slate-900` 通用渐变单块 |
| 选业态 | 找自己的行业 | 「懂我」 | 大类图标用 **单色线框 + 品牌色点缀**，不用彩虹 emoji 墙 |
| 填地址 | 怕填错 | 「可控」 | 自动完成 + 明确错误文案 |
| 选档 | 怕买错档 | 「中间档安全」 | 三列对比：**仅 5 行差异**，其余收进「展开全部」 |
| 付费前 | 犹豫 | 「值」 | 一行 **ROI 锚定**：「一次报告 &lt; 一顿团队午餐」类 copy（需合规不夸大） |
| 付费后 | 等报告 | 「快」 | 成功页明确「邮件/链接保存」+ 预计生成时间 |

**时间维度：** 5 秒内需完成「我是谁、对我有什么用、下一步点哪」。

## AI Slop 风险（Pass 4）— PRD 示例代码整改清单

PRD 中 `app/iq/page.tsx` 示例触发 **技能 Hard Rejection / 黑名单** 项：

1. **标题 emoji `🎯`** → 移除；用字重与留白做层级。  
2. **`from-slate-900 to-slate-800` 大块渐变** → 与现网 IQ 页不一致且易显模板感 → **延续** `min-h-screen` + `border-white/10` + `bg-white/5` 卡片语言。  
3. **分类用 emoji（🍜🛍️）** → 改为 **图标组件 + 中英文标签**，emoji 仅作可选装饰小点。  
4. **三档定价在 PRD 图中已是「功能矩阵」** — 上线 UI：**中间档加粗边框 + Recommended**，左右弱化；避免三等分完全对称「模板 Pricing」。

**Litmus（计划阶段自检）：**

- 首屏能否 **不看小字** 知道产品？  
- 去掉所有阴影后，**层级是否仍成立**？（靠对比度与字号）  
- **每张卡片是否必要？** 步骤容器一张即可，不要 hero 下再堆三张 feature card。

## 响应式与无障碍（Pass 6）

- **触控目标：** 业态格子最小 **44×44px**；档位整张卡片可点，不仅「选择」小按钮。  
- **键盘：** 业态网格 `roving tabindex` 或顺序 Tab；步骤条 `aria-current="step"`。  
- **对比度：** 白字 on `white/40` 仅用于脚注；**主文案 ≥ WCAG AA**。  
- **响应式：**  
  - `sm`：三档定价 → **横向滚动 snap** 或 **折叠为手风琴**，禁止三列挤成蚂蚁字。  
  - 地址栏全宽优先于侧边空白。

## 未决决策清单（Pass 7）— 需在 PRD 拍板

| 决策 | 若不定稿的后果 |
|------|----------------|
| 主品牌色与 IQ 翠绿是否统一 | 营销站与漏斗「像两个产品」 |
| **是否同时提供 Basic（$9）档** | PRD 已定 Pro 为默认；若也提供 Basic 需确认 Stripe SKU、定价与对应 tier 参数映射 |
| 多语言下 **档位名** 是否意译 | 结账页与邮件不一致 |
| 报告预览是否展示 **分数 0–100** | 影响「专业感 vs 恐吓感」 |
| 是否展示「数据来源」徽标行 | 信任提升 vs 法务审阅 |
