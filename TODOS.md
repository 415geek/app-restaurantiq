## 🏗️ 架构决策记录 (ADR)

| 决策项 | 选择 | 说明 |
|--------|------|------|
| 数据源调用 | **并行 + Promise.allSettled** | 最快响应 (~2-3秒)，部分 API 失败不影响整体 |
| 业态配置 | **代码内 JSON 配置** | `/lib/config/business-types/*.json` 版本控制 |
| 成本追踪 | **调用级明细** | `data_source_calls` 表记录每次 API 调用 |
| 失败处理 | **分级处理** | Tier 1 (Census/Google) 必须成功，Tier 2/3 可降级 |
| API 缓存 | **智能差异化 TTL** | Census=30天, Google=7天, Yelp=3天, 人流=不缓存 |
| LLM 策略 | **串行调用** | Pro+: GPT-4o 生成 → Claude 审核补充 |
| Schema 升级 | **纯增量** | 新字段允许 NULL，无历史数据迁移 |
| 发布策略 | **分阶段** | 1.DB Schema → 2.Backend API → 3.Frontend UI |
| 测试策略 | **真实 API** | 需注意 API 配额和费用控制 |

## 📋 已延后 (Deferred)

### 白标解决方案
- **What**: 允许企业客户使用自己的品牌嵌入 BusinessIQ
- **Why**: 企业级客户愿意支付 $5K-20K/年的白标费用
- **Pros**: 高价值 B2B 收入，市场扩展
- **Cons**: 需要多租户架构，品牌定制，复杂度高
- **Context**: 建议在核心产品稳定、有 10+ B2B 客户需求后再考虑
- **Effort**: XL (human) → L (CC)
- **Priority**: P3
- **Depends on**: EXT-001 (B2B API) 完成

### AI 选址对话助手
- **What**: 交互式 AI 助手，支持追问和多轮对话
- **Why**: 增加用户粘性，可作为付费附加功能
- **Pros**: 差异化功能，增强体验
- **Cons**: 不是核心价值，会增加产品复杂度
- **Context**: 可复用现有 Ops Copilot 架构
- **Effort**: M (human) → S (CC)
- **Priority**: P3
- **Depends on**: ENGINE-001, ENGINE-002 完成

## ⚠️ 风险登记簿

| 风险 | 级别 | 缓解措施 |
|------|------|---------|
| Unacast API 成本超支 | 🔴 高 | 严格限制在 Pro+ 档位调用 |
