## Learned User Preferences

- IQ reports (free and paid) should read like McKinsey-grade location consulting grounded in address-specific Yelp / Foursquare / Google Places pulls — not generic LLM-only copy, placeholder competitors, identical revenue bands across addresses, or speculative claims without retrieved market signals or real examples.
- When scoping IQ report work, treat **server PDF export** as higher priority than web-only visual polish when those goals conflict (stakeholder direction: PDF-first relative to on-screen layout).
- Break-even and safe-revenue must be computed deterministically from real cost inputs (rent + labor + COGS + AOV with cuisine-archetype turns/day) — not LLM-guessed — and any rent / area / budget the user supplies at intake must propagate into the finance model and prompt instead of being re-inferred.
- Risk-audit score bars must match displayed scores (bar length equals the shown number; pressure-style dimensions use semantic colors — high pressure reads as worse, not inverted bar length).
- IQ progress UI (free analyze and paid full-report generation) should show only product-language stage labels — never expose LLM/model names, vendor tools (n8n, Tavily, OpenAI, etc.), or internal pipeline steps in on-screen copy or user-visible errors.
- User expects IQ work to ship as production-ready on Vercel (`app.restaurantiq.ai`) when they ask to sync/deploy — not left as local-only changes.

## Learned Workspace Facts

- When `N8N_IQ_FULL_REPORT_WEBHOOK_URL` / `N8N_FULL_REPORT_WEBHOOK_URL` or `N8N_IQ_ANALYZE_WEBHOOK_URL` is set, those routes call n8n; deploy prompt/workflow changes in n8n (and repo TypeScript). Local dev with analyze webhook env set hits the configured n8n instance (often production prompts).
- `full_report_json` is cached until regeneration. `POST /api/funnel/full-report` with `force: true` rebuilds; `quality: true` (default when `force: true`) runs full premium market enrich and non-lean generation — otherwise lean shortcuts apply.
- Paid unlock paths all call **`fulfillIqPaidPurchase`** (`lib/funnel/iq-complete-purchase.ts`): Stripe `checkout.session.completed`, `/iq/success?session_id=…`, and **`POST /api/funnel/redeem-access-code`** (`IQ_ACCESS_CODE` env). Each can trigger inline full-report generation (minutes, not instant).
- Generation core: **`generateIqFullReportWithN8nFallback`** (`lib/funnel/iq-generate-full-report.ts`) — n8n when webhook URL is set, OpenAI fallback on n8n failure or missing credentials.
- Full-report route timeout: `app/api/funnel/full-report/route.ts` and `vercel.json` use **`maxDuration: 300`**. Client progress (`IqFullReportGenerating`) caps around **92%** until the API returns — failures near **89–92%** usually mean generation timeout/failure, not a progress-bar bug.
- Paid PDF: `GET /api/iq/report/{id}/pdf?lang=zh|en` — Vercel uses `@sparticuz/chromium` only; `setContent` with `waitUntil: 'load'`. UI should download the Blob; avoid surfacing vendor names in PDF error paths.
- IQ V2 prompts (`lib/funnel/iq-prompts-locationiq-v2.ts`) are cuisine-specific (hot pot/BBQ, boba/coffee, HK cafe/烧腊, QSR, etc.) for trade area, competitors, and turns/day — do not regress to generic templates.
- Free tier: `/api/funnel/analyze` prefetches via `resolveMarketDataForIqReport(..., isPremium: false)` + `buildFreeTierMarketBrief`; paywall mirrors paid sections via `getIqPaywallLockedItems`. Gated by `POST /api/iq/lead` (email + cuisine).
- Finance + competitors: `computeFinanceModel` (`lib/funnel/iq-finance-model.ts`) feeds break-even/safe revenue into prompts; real competitor quality needs `YELP_API_KEY`, `FOURSQUARE_API_KEY`, `GOOGLE_MAPS_API_KEY`. Premium optional: Tavily Deep Research (`TAVILY_API_KEY` → `market_data_json.deep_research`).
- Models: `OPENAI_IQ_MODEL` (free, default `gpt-4o-mini`) and `OPENAI_IQ_FULL_MODEL` (paid, default `gpt-4o`) in `lib/funnel/iq-llm.ts`; pipeline is OpenAI-only in code (no wired second-pass reviewer).
- Clerk/auth: `docs/CLERK_REDIRECTS.md`, `NEXT_PUBLIC_APP_URL`, guest reports link via `POST /api/iq/link-report`. Admin leads: `POST /api/admin/login` with `ADMIN_EMAIL` + `ADMIN_PASSWORD` (no public admin registration).
