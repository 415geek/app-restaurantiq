## Learned Workspace Facts

- If `N8N_IQ_FULL_REPORT_WEBHOOK_URL` or `N8N_FULL_REPORT_WEBHOOK_URL` is set, production paid full reports are produced by the n8n workflow; deploy prompt changes there, not only in repo TypeScript.
- `full_report_json` is reused until regeneration; use `POST /api/funnel/full-report` with `force: true` after prompt or data pipeline changes.
- IQ auth redirects and Clerk allow-list targets are documented in `docs/CLERK_REDIRECTS.md`; `NEXT_PUBLIC_APP_URL` drives absolute post-sign-in URLs; guest-paid reports attach to the signed-in user via `POST /api/iq/link-report` (triggered from the report page client when Clerk session exists).
- Paid IQ report PDFs: `GET /api/iq/report/{id}/pdf?lang=zh|en` — on Vercel, Puppeteer launches **only** `@sparticuz/chromium`; HTML uses `setContent` with `waitUntil: 'load'` (avoid `networkidle0` for static HTML). Local dev: set `PUPPETEER_EXECUTABLE_PATH` or `CHROME_PATH` to a Chrome binary if needed. Report UI should prefer downloading the response as a Blob and fall back to `window.print()` only if the API fails.
- **Stripe Checkout → paid report access:** `/iq/success?session_id=cs_…` verifies the Checkout Session and calls **`fulfillIqPaidPurchase`** (same core path as the `checkout.session.completed` webhook) before redirecting to `/iq/report/{id}`, so `paid` is set even when webhooks are delayed or misconfigured—do not assume webhooks alone unlock the report on first load.
- Paid full report generation entry point: **`generateIqFullReportWithN8nFallback`** in `lib/funnel/iq-generate-full-report.ts` (n8n when `getFullReportWebhookUrl()` is set, with OpenAI fallback on n8n failure or Class A config gaps like missing credentials); **`fulfillIqPaidPurchase`** in `lib/funnel/iq-complete-purchase.ts` centralizes marking paid + generating `full_report_json` for both the webhook and the success URL.
- Premium IQ reports use Tavily Deep Research API (`TAVILY_API_KEY`) for McKinsey-style location analysis; default model is "pro" with 300s timeout; results stored in `market_data_json.deep_research`.
- IQ V2 prompt system at `lib/funnel/iq-prompts-locationiq-v2.ts` includes a cuisine-specific framework (hot pot/BBQ, boba/coffee, HK cafe/烧腊, QSR, etc.) that tunes trade-area radius, competitor identification, and turns-per-day assumptions — do not regress to generic single-cuisine templates.
- Free-tier `/api/funnel/analyze` pre-fetches market data via `resolveMarketDataForIqReport(..., isPremium: false)` (Places + ACS, no Deep Research) and injects a compact anchor with `buildFreeTierMarketBrief` from `lib/funnel/iq-premium-anchors.ts` before OpenAI/n8n; paywall UI mirrors paid sections via `getIqPaywallLockedItems` in `lib/funnel/iq-paywall-sections.ts`.
- IQ LLM models are env-tunable: `OPENAI_IQ_MODEL` (free, defaults `gpt-4o-mini`) and `OPENAI_IQ_FULL_MODEL` (paid, defaults `gpt-4o`) in `lib/funnel/iq-llm.ts`; current pipeline is OpenAI-only — no second-pass reviewer model is wired in code despite older planning docs implying one.
- ACS demographic data enrichment requires geocode from Google Maps API (`GOOGLE_MAPS_API_KEY`) to convert addresses to lat/lng for census tract lookups.
- Caltrans AADT traffic data API (`lib/funnel/external-data/caltrans.ts`) only covers California state highways (101, 280, etc.), not local streets; uses ArcGIS REST API at `caltrans-gis.dot.ca.gov`.
- Optional commercial-data integrations behind env keys: `LOOPNET_RAPIDAPI_KEY` (commercial listings via RapidAPI — falls back to manual search instructions when absent); `BRIGHTDATA_API_TOKEN` (Bright Data MCP for Google SERP, Zillow, Maps reviews, LinkedIn, Crunchbase).
